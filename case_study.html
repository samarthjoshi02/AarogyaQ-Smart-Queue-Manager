🏥 AarogyaQ Case Study

Smart Queues for Healthier Waiting

Replacing paper hospital tokens with a real-time clinical waiting room and database spreadsheet orchestrator

1. Executive Summary

AarogyaQ is a fullstack clinic queue manager designed to optimize hospital patient flows. By utilizing real-time WebSockets, a lightweight persistent datastore, and background compilers, the system bridges the gap between receptionist administration, public patient notifications, and digital medical records.

2. The Problem

In typical community health clinics, traditional paper-slip queuing systems lead to crowded waiting rooms, patient anxiety, and increased transmission of infections. Receptionists face a high operational load: they must log walk-ins, manually write token slips, manage patient queries, and manually cross-check lists to calculate waiting averages. This chaotic process is prone to bottlenecks and spreadsheet errors.

3. The Solution & Workflow Design

AarogyaQ condenses the entire consultation administrative loop into a single action. By shifting the workload to a smart container model, the receptionist dashboard enables queue controls, checkup settings, and analytics under a unified interface.

Before (Manual Setup - 8 Taps)
  • Open local tracking spreadsheet file.
  • Manually write patient name, age, and phone.
  • Write down token number onto paper.
  • Hand paper to patient.
  • Paging patients manually via voice/shouting.
  • Use Excel formulas to calculate wait averages.
  • Manually save spreadsheet log file.
After (AarogyaQ - 1 Tap)
  • Click "Call Next" on dashboard controls.
  • Node backend auto-updates SQLite logs.
  • SheetJS auto-compiles Excel spreadsheet.
  • Socket.IO streams updates to all screens instantly.
  • Web Audio synth chimes play caller alerts.
  • Responsive SVG graphs display analytics.

4. Core Technical Architecture

  • Frontend Interface: Built with React, structured on a modern HSL styling system using vanilla CSS. Custom responsive SVG Area and Bar charts allow analytics to render cleanly on React 19 without dependency warnings.
  • Real-time State Server: An Express server configured with Socket.IO that streams live states (Now Serving, total waiting, analytics averages) in under 10ms.
  • Persistent Database: SQLite3 tables manage patient data, consultation speed configurations, and records.
  • Direct Office Compilation: A SheetJS (`xlsx`) script matches database events to output a beautifully formatted Excel file (AarogyaQ_Queue_Data.xlsx) directly in the directory.

5. Product Screen Breakdown

The system features three primary modules:

  • Receptionist Dashboard: Contains quick-action controls, settings to change consultation times, a dynamic sortable queue table, and real-time analytical charts.
  • Patient Personal Portal: Features simulated Google and phone OTP auth drawers. Patients can book slots, save their details locally, and watch their queue position advance in real-time.
  • TV Waiting Room Display: Optimized dark midnight-blue screen for hospital monitors. Emits chimed sound announcements and features a scrolling helpline news ticker.

6. Outcomes & Self-Awareness Reflection

During testing, the system successfully managed concurrent connections, updating waiting room lists instantly. Serving a patient and syncing logs was optimized from 8 taps down to a single click.

Future Scope & Improvements: Given more time, the mock auth flows would be upgraded to production-ready Twilio/Firebase APIs. An SMS integration would be added to alert patients when they are three slots away, allowing them to wait elsewhere. The system would also be expanded to support multiple doctor rooms rather than a single consultation queue.
